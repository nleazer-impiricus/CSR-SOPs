Detailed SOP Steps

Step 1 — Review the Ticket

Read the full message thread. Understand the HCP's request and any relevant history before taking action.

💡
Maxima AI (Zendesk app) may be used to assist with reviewing HCP history and ticket context.

Step 2a — Search the Knowledge Base All Groups

Search the Zendesk KB for a verified sample process for this product (brand and generic name).

🔀Process found in KB?
Yes → Follow it and proceed to Step 3.
No → Continue to Step 2b.

Step 2b — Search Slack All Groups

Search both #concierge-questions and #concierge-updates for a previously approved process for this product.

🔀Coordinator-approved process found?
Yes → Follow it and proceed to Step 3. If the process exists in Slack but not in the KB, flag the gap in #concierge-questions so it can be added.
No → Proceed to Step 2c (Perplexity Research).

Step 2c — Research with Perplexity AI Group 1 Only

Use the approved Perplexity prompt to research a process for this product. See the Perplexity Guide and Prompt Template tabs for full instructions.

🚩
Verify all findings before proceeding. Never send unverified information to an HCP. Only U.S. manufacturer or HCP-facing sources are acceptable. Remove all tracking parameters from URLs.
⚠️
Novel processes require Coordinator approval before you reply to the HCP. Post in #concierge-questions and wait for sign-off.

Step 3 — Log All Actions in Internal Notes All Groups

Before replying to the HCP, document everything in an internal note:

  • Emails, phone numbers, and rep names contacted
  • URLs, portals, or forms used
  • Screenshots of any submissions or confirmations
  • Any Coordinator approval received
🚩
Always verify you are saving to an internal note — not a public reply — before submitting.

Step 4 — Send Findings to the HCP All Groups

Send the HCP a message summarizing findings and what to expect next. Use an approved macro when available.

📨
All HCP messages must be sent via the Sunshine Conversations API "send" option (within the text entry box). Abide by the HCP's communication preferences as closely as possible.

Step 5 — Next Steps Based on Outcome All Groups

🔀What happens next?
New or different request → Flag to your Coordinator for a new ticket or HCP reassignment. Do not work the new request on the existing ticket.
Additional follow-up needed → Keep the ticket open. Update both the HCP and internal notes each time the ticket status changes.
Conversation complete → Apply the closing macro or survey macro and close the ticket.

Closing macro: use when HCP has previously received the survey.
Closing macro with survey: use if the HCP has not previously received the survey.

📌 Ongoing Notes

  • If you receive new HCP information (address, phone, preferred contact method), add it to the HCP's user profile in Zendesk.
  • Do not send AI-generated output directly to an HCP — all Perplexity findings must be verified first.
  • When in doubt: stop and escalate to your Coordinator before replying to the HCP.
  • Ticket status should be updated anytime there is a new action or change.

Perplexity AI — What It Is & How to Use It

What is Perplexity?

Perplexity is an AI-powered search engine. Unlike ChatGPT (which generates answers from memory), Perplexity actually searches the internet in real-time and shows you the sources it found. Think of it like Google, but instead of giving you a list of blue links, it reads the pages for you and writes a summary with citations.

Why are we using it?

For our use case — finding manufacturer sample programs, rep contacts, and HCP resources — Perplexity is helpful because it can search multiple sites at once and pull together information that would take several Google searches to find manually.

The Critical Rule: Always Verify

Perplexity (and all AI tools) can and will make mistakes. It may return links that don't exist or confidently describe programs that aren't real. You are responsible for verifying every piece of information before it goes to an HCP.

🚫
Never copy-paste Perplexity's output directly to an HCP. Always click every link, verify the process exists on the manufacturer's actual website, and complete the Final Verification Checklist before posting to Slack.

Getting Started (One-Time Setup)

  1. Go to perplexity.ai and create a free account.
  2. Once signed in, you'll see the search bar — this is where you'll run prompts.
  3. Save the prompt from the Prompt Template tab as a Perplexity template.
  4. To save: click "Library" → "Prompts" → "Create Prompt" → paste the prompt → set shortcut /samples.
  5. Now anytime you need it, type /samples in the search bar.

Running a Search

  1. Open Perplexity and click the search bar.
  2. Type /samples — your saved template will appear. Select it.
  3. Replace [drug name] with the medication you're researching.
  4. Press Enter. Perplexity will search the web and return a structured answer.
  5. Review the answer — check each cited source by clicking the numbered references.
  6. Verify every link and claim before posting to Slack for approval.
💡
Tip: If Perplexity's first answer is incomplete, ask follow-up questions in the same thread — e.g.: "Can you also check for a demo trainer or device request form?"

🛡️ Final Verification Checklist

Complete before posting to #concierge-questions — regardless of how you found the information.

🛡️ All Items Must Pass Before Posting to Slack

  • U.S. manufacturer site: Confirm the website is the U.S. version (not .co.uk, .ca, .eu). Check footer/header.
  • Forms accept U.S.: Verify "United States" is available, accepts NPI, zip codes, state selections.
  • Every link works: Click every URL. Does it load? Does it go where expected?
  • Program exists: Confirm the sample/rep/demo program is actually described on the official site.
  • Information is current: Check for copyright dates, last-updated notes, discontinuation notices.
  • All sources are official: Only manufacturer websites, FDA.gov, or official HCP portals.
  • Clean URLs: Remove tracking parameters (?utm_source=chatgpt.com, ?ref=). Test the cleaned URL.

Perplexity Prompt Template

Copy this entire prompt and save it as a Perplexity template with the shortcut /samples. Replace [Drug Name] with the medication you're researching.

SYSTEM ROLE You are a U.S.-based pharmaceutical commercial sampling research specialist. The user provides only a drug name. Operate under U.S. market assumptions unless otherwise stated. Your mandate is to verify whether licensed U.S. healthcare professionals (HCPs) can obtain official manufacturer samples, demo kits/trainers, rep visits, bridge programs, phone/fax requests, or chatbot-routed access. Identify the official access pathway and confirm the responsible commercial entity. PRIORITY ORDER (MANDATORY): Samples > Demo Kits > Reps > Bridge Programs > Savings Cards Prioritize accuracy over completeness. Base all conclusions only on verifiable manufacturer, HCP, or FDA sources. HCP SITE PRIORITIZATION RULE (MANDATORY OVERRIDE) Weigh HCP-specific website content 10x higher than patient-facing content. HCP site indicators: /hcp, /pro, /professional, /medical, /physician, hcp.* subdomain, or "For Healthcare Professionals" disclaimers. PrescriberPoint.com Rule: Treat prescriberpoint.com as a third-party aggregator. Reduce confidence by 15%. Display only in master table with asterisk. If both patient and HCP websites exist, ignore patient site entirely unless HCP version is absent. CORPORATE PORTAL HEURISTIC (CRITICAL OVERRIDE) MANDATORY portals to check: MerckSamplePortal, MySampleCloset, KenvuePro, PfizerPro, NovoMedLink, AbbViePro, LillyTrade, AZMedical, JanssenPro, BauschHealth, Intra-Cellular, bdf.hcpsampling.com. PHASE 0 - DRUG TYPE CLASSIFICATION (MANDATORY) 1. Run: "[Drug] generic OR OTC OR over counter OR brand name" 2. If OTC: Proceed normally 3. If Generic: Flag ALL pathways with "Generic, sampling programs may have ended" PHASE 1A - SPECIALTY DETECTION (MANDATORY) 1. "[Drug] indications FDA" 2. Extract all U.S. FDA-approved indications 3. Generate specialty-specific queries (Derm, GI, Rheum, Onc) 4. Run 1-2 top queries per specialty (max 3 total) PHASE 1 - EXPANDED EVIDENCE SEARCH Run these searches (quotation marks mandatory): "[Drug] HCP sample request" "[Drug] demo trainer HCP" "[Drug] request rep HCP" "[Drug] phone samples HCP" "[Drug] fax samples request" "[Drug] chatbot rep samples" "[Drug] MySampleCloset OR mysamplecloset.com" "[Drug] request trainer device" "[Manufacturer] HCP phone number" "[Drug] FDA label PDF" "[Drug] distributed by" PHASE 1B - SITEMAP SCAN (MANDATORY) Run: "[Drug] HCP site-map" OR "[Drug] HCP sitemap" Flag ALL paths matching: /contact-a-rep, /request-samples, /hcp/trainer, etc. PHASE 2 - HCP PORTAL DEEP SCAN Visit corporate portals + drug HCP sites. Navigate PER SPECIALTY. Extract: Phone/fax numbers, chatbot domains, demo trainer forms, bridge programs. PHASE 3 - GRADING Grade A: Direct sample/demo trainer ordering Grade B: Phone/fax requests, chatbots, bridge programs Grade C: Rep request forms mentioning samples Grade D: Rep request forms only Grade E: No evidence found PHASE 4 - Extract Verifiable Facts Record: Portal URLs, phone/fax numbers, chatbot domains, demo trainer links, bridge eligibility, sample limits. PHASE 5 - Confidence Scoring Start 100% then deductions: login barriers (-15%), phone-only (-10%), rep dependency (-20%). PHASE 6 - MASTER OUTPUT TABLE | Pathway Rank | Type | Company | Link/Phone | Steps | What's Offered | Grade | Confidence | FINAL DETERMINATION Result: ✅ Confirmed / ❌ No program found / ⚠️ Insufficient evidence LINKING RULES: All links must begin with https://. Accept only manufacturer/HCP domains. Cross-check WHOIS/footer branding. INFERENCE PROHIBITION: Never infer from competitors/analogs. Confirm only explicit manufacturer/HCP sources.

Search Tips — Finding HCP Sites

Recommended Search Queries

Replace [Drug] with the medication name.

Looking ForSearch Query
HCP website[Drug] HCP site or [Drug] for healthcare professionals
Manufacturer[Drug] manufacturer
Generic availability[Drug] generic available
Demo/trainer device[Drug] demo trainer device HCP

How to Identify the HCP Version

  • URL clues: /hcp, /pro, /professional, /medical, or subdomains like hcp.drugname.com
  • Site toggle: Look for "For Healthcare Professionals" link in the header
  • Confirmation gate: HCP sites often ask you to confirm you're a licensed professional

🇺🇸 Confirming It's a U.S. Site

  • Check URL: Avoid .co.uk, .ca, .eu, .com.au
  • Check footer: Look for "United States" in country selector
  • Check header: Country/region selector set to U.S.
  • Check forms: Accepts U.S. NPI, state, zip codes

What to Look For on the HCP Site

  • Resources / Support: Sample request forms, starter kits, bridge programs
  • Contact / Request a Rep: Forms to request a sales representative visit
  • Getting Started: Often contains sample info, onboarding materials
  • Footer links: Phone numbers for medical information (800 numbers)
✅
Trust only official manufacturer domains. If you find info on a third-party site, go to the manufacturer's actual website to confirm.
